```Javascript
var a = {n : 1};    
var b = a;  
a.x =a = {n : 2};  
console.log(a.x);    // undefined
console.log(b.x);    // { n:2 }
```
## 解析
### JS引用类型指针的工作方式
&emsp;&emsp;首先是a指向了一个对象｛n:1｝,b指向了a所指向的对象
&emsp;&emsp;然后 **a.x = a = { n:2 }**
&emsp;&emsp;js赋值运算顺序为从右往左，不过由于"."运算符优先级最高，所以先计算a.x。此时a指向的对象｛n:1 }新增了新的属性x,这个x的值为undefined。
&emsp;&emsp;接着执行赋值预算 **a={n:2}**,这个时候a指向的对象就发生了变化，指向了新的对象 ｛n:2｝。
&emsp;&emsp;继续执行 **a.x = a**。此时啊a.x已经指向了对象｛n:1,x=undefined}中的x,等待被赋值，那么这个对象就变成了｛n:1,x:{n:2}｝,也就是b所指向的对象。

### 本质在于   .运算优先于=赋值运算
